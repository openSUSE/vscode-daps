DAPS extension for VSCode #

 
Publication Date: 2026-02-20

1 Introduction #

 

1.1 Features #

 

The DAPS tool helps you author and publish documentation written in DocBook XML and AsciiDoc. This extension makes it easier to run selected DAPS commands from the VSCode editor environment.

1.2 Requirements #

 

This extension requires that you…​

  • install DAPS on your system. Refer to https://opensuse.github.io/daps/ for more details.

  • install the AsciiDoc extension to VSCode

  • install this DAPS extension to VScode

1.3 Known Issues #

 

See our issue tracker at openSUSE/vscode-daps/issues.

2 Configuration #

 

The vscode-daps extension offers many options that you can specify in the settings.json file to customize and fine-tune its behavior. To view and modify all available options, press Ctrl–, and click Extensions › DAPS. Alternatively, customize the options directly in settings.json.

Tip: Task-specific options

Each relevant task in this guide has a section called Relevant options. It describes options that can affect performing the specific task.

Tip: Making it quicker with options

If you work on a project and use the same DC file, target format or root ID over and over, you can specify options in the .vscode/settings.json file of your project. After you save the modified .vscode/settings.json file, you will not be asked by VSCode to specify the included options when validating or building running documentation.

3 Markup-independent commands #

 

The following sections describe commands that you can perform on both DocBook and AsciiDoc documents.

3.1 Autocompleting entities/attributes #

 

For the autocomplete to work, the active document must include entity/attribute definitions such as

:osuse:   "openSUSE" # for AsciiDoc
<!ENTITY osuse   "openSUSE"> # for DocBook

or include a file with entities/attributes similarly to

include::../common/attributes.adoc # for AsciiDoc
<!ENTITY % entities SYSTEM "../common/generic-entities.ent"> # for DocBook

Start typing { (AsciiDoc) or & (DocBook) and the editor tries to find the right attribute as you type.

3.1.1 Relevant options #

 
daps.autocompleteXMLentities

If enabled, import XML entities from the active and externally referenced files. Default is true.

daps.excludeXMLentityFiles

Exclude specified entity files from parsing. For example:

"daps.excludeXMLentityFiles": [
  "network-entities.ent"
]

3.2 Replacing entities/attributes #

 

For the replacement to work, the active document must include entity/attribute definitions such as

:osuse:   "openSUSE" # for AsciiDoc
<!ENTITY osuse   "openSUSE"> # for DocBook

or include a file with entities/attributes similarly to

include::../common/attributes.adoc # for AsciiDoc
<!ENTITY % entities SYSTEM "../common/generic-entities.ent"> # for DocBook

Snippets in the document’s text that match one or more attribute/entity definitions are underlined. If you click inside such snippet and press Ctrl–., you can select the attribute/entity to replace the snippet with.

The replacement suggestions are also listed in the PROBLEMS panel and you can perform the replacement there.

3.2.1 Relevant options #

 
daps.replaceWithXMLentity

If enabled, automatically suggests to replace strings with their corresponding XML entities. Default is false.

daps.replaceWithADOCattribute

If enabled, automatically suggests to replace strings with their corresponding AsciiDoc attributes. Default is false.

daps.autocompleteXMLentities

If enabled, import XML entities from the active and externally referenced files. Default is true.

daps.excludeXMLentityFiles

Exclude specified entity files from parsing. For example:

"daps.excludeXMLentityFiles": [
  "network-entities.ent"
]
daps.replaceWithIgnorePhrases

A list of phrases that will be skipped from replacing with an entity or attribute if matched. For example:

"daps.replaceWithIgnorePhrases": [
  "users",
  "help",
  "reference",
  "article"
]
daps.replaceWithXMLentityIgnoreTags

A list of XML tags inside which entity replacement suggestions should be ignored. For example:

"daps.replaceWithXMLentityIgnoreTags": [
  "screen",
  "command",
  "package",
  "option",
  "filename",
  "systemitem",
  "literal",
  "guimenu",
  "envar",
  "remark",
  "quote",
  "replaceable"
]
daps.replaceWithADOCattributeIgnoreBlocks

A list of AsciiDoc block styles inside which attribute replacement suggestions should be ignored. For example:

"daps.replaceWithADOCattributeIgnoreBlocks": [
  "source",
  "literal",
  "listing",
  "passthrough",
  "screen",
  "code",
  "console"
]

3.3 Validating a document #

 

There are several ways to validate a document:

3.3.1 From the Explorer view #

 

Right-click the DC file in the Explorer view and select DAPS › Validate.

3.3.2 From the Command Palette #

 

  1. Press Ctrl–Shift–P to open the command palette.

  2. Start typing daps and select DAPS: Validate.

  3. From the populated drop-down list, select the DC file to validate.

3.3.3 From DC file’s editor tab #

 

  1. Right-click the tab of the editor showing a DC file content.

  2. Select DAPS › Validate.

3.3.4 Relevant options #

 
daps.enableExtendedValidation

Additional level of validation. Leave empty to keep the default validation level, or specify comma-separated combination of ids, images, inlines or tables. all enables full validation.

3.4 Exporting a document #

 

There are several ways to export a document:

3.4.1 By specifying a DC file from Explorer #

 

To build a document specified by its DC file, right-click the DC file in the Explorer view and select DAPS › Build with DC file. When asked, choose whether to build pdf or html.

3.4.2 By specifying a DC file from the Command Palette #

 

If you prefer typing to mouse movements, you can run the build command from the Command Palette:

  1. Press Ctrl–Shift–P and start typing daps.

  2. When available, select the DAPS: Build with DC file command.

  3. Select the DC file from the populated list and choose whether to build pdf or html.

Tip

After the build is finished, you can directly open the resulting document or copy its path to a clipboard.

3.4.3 By specifying a top-level ID from the Command Palette #

 

Besides building a whole book, you can build only a part of it, for example, a chapter or a topic. To achieve this, you need to specify a top-level element ID:

  1. Press Ctrl–Shift–P and start typing daps.

  2. Select the DAPS: Build with root ID command.

  3. Select the DC file from the populated list, then the root ID to build.

  4. Finally, choose whether to build pdf or html.

Important
Populating the list of root IDs can take a few seconds.

3.4.4 By specifying a top-level ID from the active editor #

 

  1. Select the top-level ID by the mouse and right-click the selection.

  2. Select DAPS › Build with root ID.

  3. Select the DC file from the populated list and choose whether to build pdf or html.

3.4.5 Relevant options #

 
daps.buildRootId

Set the top-level ID of a document to build. If not specified, the user is asked to enter it.

daps.DCfile

Set the DC file specific to the current workspace. If not specified, the user is asked to select one.

daps.buildTarget

The output format (html or pdf)specific to the current workspace. If not specified, the user is asked to select one.

3.5 Checking external links #

 

You can check whether external http(s) links in the active document are valid. If a link is not valid, it is reported in the PROBLEMS panel. If an http link has a valid https variant, it is suggested in the PROBLEMS panel as well.

  1. Press Ctrl–Shift–P and start typing daps.

  2. Select the DAPS: Check for broken links command.

  3. Check the PROBLEMS panel for reports of broken links. These are underlined within the open document.

3.5.1 Relevant options #

 
daps.linkCheckIgnoreList

Ignore checking URLs that include these phrases. For example:

"daps.linkCheckIgnoreList":
[
  "localhost",
  "example"
]
daps.linkCheckOnSave

Check for broken links when a file is saved. If enabled, all links in the active document are check on each save which may waste system resources. Default is disabled which means that you need to invoke the link check manually by running the DAPS: Check for broken links command.

4 DocBook-specific commands #

 

The following sections describe commands that you can perform on DocBook documents.

4.1 Formatting a document #

 

To format the document in the active editor, right-click its tab and select DAPS › Format XML Document.

Tip

Use the predefined Ctrl–Shift–I] shortcut to format a document.

Tip

To format any (even closed) document, right-click its name in Explorer and select DAPS › Format XML Document.

4.1.1 Relevant options #

 
daps.XMLformatExecutable

Specify the path to the xmlformat executable. Default is daps-xmlformat.

daps.XMLformatConfigFile

Specify the path to `xmlformat’s configuration file.

4.2 Exporting a single file #

 

To build the document in the active editor, right-click its tab and select DAPS › Build XML file. Finally, choose whether to build pdf or html.

To build any (even closed) file, right-click its name in the Explorer and select DAPS › Build XML file. Finally, choose whether to build pdf or html.

4.3 Previewing a document #

 

To preview the document in the active editor, right-click its editor tab and select DAPS › Preview Document. A new tab opens to the right of the active editor with the preview. It is updated on each source editor change.

4.3.1 Relevant options #

 
daps.docPreviewImgPath

Path to images relative to the document. For example:

"daps.docPreviewImgPath": "../images/",
daps.previewLinkScrollBoth

Whether scrolling the preview scrolls the source editor as well. Default is false.

4.4 Viewing a document outline #

 

You can view a document outline in the Explorer pane.

Tip

Navigate to a displayed section by clicking its title in the structure tree.

4.4.1 Relevant options #

 
daps.onClickedStructureItemMoveCursor

After clicking an item in the document structure, whether to show the specific line only or move the cursor there as well. Default is true.

daps.structureElements

Which elements should be included in the document outline. For example:

"daps.structureElements":
  [
    "section",
    "sect1",
    "sect2",
    "sect3",
    "sect4",
    "procedure",
    "example"
  ]

4.5 Peeking into a referenced document #

 

You can peek into or open referenced files directly from the referencing file.

4.5.1 Relevant options #

 
daps.dbAssemblyPattern

A glob pattern to identify DocBook assembly files. Default is "*/.asm.xml"

daps.openFileSplit

Whether open a new file from CodeLens in a new tab or split window. Default is false.

daps.showAssemblyCodelens

Whether and which links to display for DocBook assembly included files. Can be one of peek, link, both or disabled. Default is disabled.

daps.showXIncludeCodelens

Similar to daps.showAssemblyCodelens but for non-assembly files that include other files. Default is disabled.

daps.showXrefCodelens

Similar to daps.showAssemblyCodelens but for files referenced by <xref/> links. Default is disabled.

daps.xrefCodelensExcludeDirs

Directories to exclude when searching for targets used in codelens. For example:

"daps.xrefCodelensExcludeDirs":
  [
    "build",
    "images",
    "l10n",
    "templates"
  ]
